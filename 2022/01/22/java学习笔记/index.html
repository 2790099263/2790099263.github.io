构造方法

每个类都有自己的构造方法，如果我们没有定义特定的构造方法，Java会使用默认的构造方法，一个类可以有多个构造方法，构造方法必须和类同名

Java基本数据类型

Java有两大数据类型

  • 内置数据类型
  • 引用数据类型

内置数据类型

  • byte 八位有符号 数据范围 -128~127 默认值为0
  • short 十六位有符号，数据范围-32768~32767 默认0
  • int 三十二位有符号 数据范围 -2^31 ~ 2^31-1
  • long 六十四位有符号 数据范围 -2^63 ~ 2^63-1
  • float double char 都参考c++
  • boolean 类比c++中的bool

对象·数组都是引用数据类型，虽有引用类型的默认值都是null

基本数据类型和引用数据类型的比较

基本数据类型在栈中进行分配，分配速度较快，变量名指向具体的数值，变量名声明之后Java会立即分配内存数据给他，并且类型之间的赋值是创建新的拷贝，==和！=是进行值的比较，基本数据类型建立和销毁比较快

引用数据类型在堆中进行分配，堆的读写速度慢于栈，变量名指向存储数据对象的内存地址，即变量名指向hash值，声明时候不会分配内存，只是存储了一个内存地址，对象之间的赋值只是传递引用，==和！=是比较两个引用是否相同，需要自己实现equals（）函数，类对象需要调用JVM进行销毁。

类型转换

自动类型转换

整型，常量，字符型数据可以进行混合运算，在运算中，不同类型的数据先转化为同一类型，然后进行运算。

数据类型转换满足以下要求

  • 不能对Boolean类型进行类型转换
  • 不能把对象类型转换成不相干的对象
  • 在把容量大的类型转化为容量小的类型时候必须使用强制类型转换。
  • 转换过程中可能会出现溢出或者损失精度。
  • 浮点数转化为小数是通过舍弃小数得到的，而不是四舍五入

强制类型转换

条件：转换的数据类型必须是兼容的。

Java实例变量

  • 实例变量声明在一个类中但是在方法，构造方法和语句块之外；
  • 当一个对象被实例化之后，每个变量的值就都跟着确定
  • 实例变量在对象创建的时候创建，在对象被销毁的时候被销毁
  • 实例变量的值应该至少被一个方法，构造方法或者语句块应用，是的外部能够通过这些方式获取实例变量信息；
  • 实例变量可以成名在使用前或者使用后
  • 访问修饰符可以修饰实例变量
  • 实例变量对于类中的方法，构造方法或者语句块是可见的，一般情况下应该把实例变量设为私有，通过使用访问修饰符可以使实例变量对子类可见
  • 实例变量具有默认值
  • 实例变量可以直接通过变量名访问。但在静态方法或者其他类中，就应该使用完全限定名ObejectReference.VariableName
