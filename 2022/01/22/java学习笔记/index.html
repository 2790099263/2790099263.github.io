Java基本数据类型

Java有两大数据类型

  • 内置数据类型
  • 引用数据类型

内置数据类型

  • byte 八位有符号 数据范围 -128~127 默认值为0

  • int 三十二位有符号 数据范围 -2^31 ~ 2^31-1
  • long 六十四位有符号 数据范围 -2^63 ~ 2^63-1
  • float double char 都参考c++
  • boolean 类比c++中的bool

基本数据类型和引用数据类型的比较

基本数据类型在栈中进行分配，分配速度较快，变量名指向具体的数值，变量名声明之后Java会立即分配内存数据给他，并且类型之间的赋值是创建新的拷贝，==和！=是进行值的比较，基本数据类型建立和销毁比较快

引用数据类型在堆中进行分配，堆的读写速度慢于栈，变量名指向存储数据对象的内存地址，即变量名指向hash值，声明时候不会分配内存，只是存储了一个内存地址，对象之间的赋值只是传递引用，==和！=是比较两个引用是否相同，需要自己实现equals（）函数，类对象需要调用JVM进行销毁。

类型转换

自动类型转换

整型，常量，字符型数据可以进行混合运算，在运算中，不同类型的数据先转化为同一类型，然后进行运算。

数据类型转换满足以下要求

  • 不能对Boolean类型进行类型转换
  • 不能把对象类型转换成不相干的对象
  • 在把容量大的类型转化为容量小的类型时候必须使用强制类型转换。
  • 转换过程中可能会出现溢出或者损失精度。
  • 浮点数转化为小数是通过舍弃小数得到的，而不是四舍五入

Java实例变量

  • 实例变量声明在一个类中但是在方法，构造方法和语句块之外；
  • 当一个对象被实例化之后，每个变量的值就都跟着确定
  • 实例变量在对象创建的时候创建，在对象被销毁的时候被销毁
  • 实例变量的值应该至少被一个方法，构造方法或者语句块应用，是的外部能够通过这些方式获取实例变量信息；
  • 实例变量可以成名在使用前或者使用后
  • 访问修饰符可以修饰实例变量
  • 实例变量对于类中的方法，构造方法或者语句块是可见的，一般情况下应该把实例变量设为私有，通过使用访问修饰符可以使实例变量对子类可见
  • 实例变量具有默认值
  • 实例变量可以直接通过变量名访问。但在静态方法或者其他类中，就应该使用完全限定名ObejectReference.VariableName

Java修饰符

Java提供的修饰符主要有两类：

  • 访问修饰符
  • 非访问修饰符

修饰符是用来定义类，方法或者变量，通常放在语句的最前端

访问控制修饰符

……感觉可以理解为关键字

Java中可以使用访问控制修饰符来保护对类，变量，方法和构造方法的访问。Java支持四种不同的访问权限。

  • default（默认，什么也不写）：可以不使用任何修饰符，这个就是默认的修饰符
  • private：在同一个类内可见。使用的对象：变量，方法，不能修饰类（外部类）
  • public：对所有类都可见。使用对象：类，接口，变量，方法
  • protected：对同一包内的类和所有子类可见，使用对象：变量，方法。（不能修饰类）

默认访问修饰符

不使用任何关键字

私有访问修饰符-private

声明变量私有，不能被外部类访问，private声明的方法，变量和构造方法只能被所属类访问，并且类和接口不能声明为private。声明为私有访问类型的变量只能通过类中公共的getter方法被外部类访问。private访问修饰符的使用主要用来隐藏类的实现细节和保护类中的数据。

公有访问修饰符-public

被声明的类，方法，构造方法和接口可以被其他任何类访问

如果几个互相访问的public类分布在不同的包中，则需要导入相应的public类所在的包。

由于类的继承性，类所有的公有方法和变量都能被其子类继承。

受保护的访问修饰符-protected（**此坑待填**）

protected：

  • 子类和基类在同一包中：被声明为protect的变量，方法和构造器能被同一个包中的任何其他类访问。
  • 子类和基类不在同一个包中：在子类中可以访问其从基类中继承的protected方法，而不能访问基类实例的protected方法

访问控制和继承

注意以下方法继承的规则：

  • 父类中声明为 public 的方法在子类中也必须为 public。
  • 父类中声明为 protected 的方法在子类中要么声明为 protected，要么声明为 public，不能声明为 private。
  • 父类中声明为 private 的方法，不能够被子类继承。

非访问修饰符

static修饰符，用来修饰方法和类变量

final修饰符，用来修饰类，方法和变量，final修饰的类不能被继承，修饰的方法不能被继承类重新定义，修饰的变量为常量，是不可修改的

abstract修饰符，用来创建抽象类和抽象方法。

synchronized和volatile修饰符，主要用于线程的编程。

static修饰符

  • 静态变量 用来声明独立于对象的静态变量
  • 静态方法 声明独立于对象的静态方法。静态方法不能使用类的非静态变量。

final修饰符

父类中的 final 方法可以被子类继承，但是不能被子类重写。

声明 final 方法的主要目的是防止该方法的内容被修改。
